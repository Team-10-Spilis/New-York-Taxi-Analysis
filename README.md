# New York Taxi Analysis

Командный проект по анализу данных, очистке, feature engineering, машинному обучению, кластеризации и визуализации данных о поездках жёлтого такси в Нью-Йорке.

> Статус проекта: активная разработка.  
> В первые дни структура репозитория, ноутбуки, скрипты и инструкции по запуску могут изменяться.

## Описание проекта

Цель проекта — проанализировать данные о поездках такси в Нью-Йорке, подготовить очищенный датасет, построить ML-модели и создать веб-интерфейс для демонстрации основных результатов.

В проект входят следующие направления:

- разведочный анализ данных;
- очистка данных и поиск аномалий;
- добавление погодных признаков;
- анализ географических зон такси;
- кластеризация зон и паттернов поездок;
- прогнозирование спроса;
- прогнозирование стоимости поездки;
- подготовка сайта для представления результатов.

## Источник данных

Основной датасет основан на данных о поездках жёлтого такси в Нью-Йорке.

Данные о поездках содержат информацию о:

- времени посадки и высадки;
- зоне посадки и зоне высадки;
- расстоянии поездки;
- количестве пассажиров;
- тарифном коде;
- типе оплаты;
- базовой стоимости поездки;
- налогах, чаевых, дорожных сборах и дополнительных платежах;
- итоговой стоимости поездки.

Также используются файлы taxi zones, которые позволяют связать идентификаторы зон посадки и высадки с реальными географическими районами Нью-Йорка.

Дополнительно в проект добавляются погодные данные: температура, осадки, снегопад и другие признаки, которые могут влиять на спрос и стоимость поездок.

## Структура репозитория

Структура проекта пока дорабатывается. В процессе разработки некоторые папки и файлы могут быть переименованы или перенесены.

## Основные части проекта

### `data clean/`

Папка содержит скрипты и ноутбуки, связанные с подготовкой данных.

Основные задачи:

- проверка пропусков;
- поиск некорректных поездок;
- удаление или исправление аномалий;
- проверка денежных признаков;
- добавление погодных данных;
- подготовка итоговых очищенных датасетов для EDA и ML.

### `clusterization/`

Папка содержит эксперименты по кластеризации.

Планируемые направления:

- кластеризация зон такси по координатам;
- кластеризация зон с учетом спроса и стоимости поездок;
- добавление временных и графовых признаков;
- подбор разного количества кластеров;
- оценка качества кластеризации;
- визуализация кластеров на карте или в двумерном пространстве.

### `ml_demand/`

Папка содержит эксперименты по прогнозированию спроса.

Основная цель:

```text
предсказать количество поездок по зоне и времени
```

Планируемые задачи:

- построить корректную агрегацию по всем зонам и всем часам;
- заполнить отсутствующие комбинации `zone × hour` нулевым спросом;
- зафиксировать общий train/test split для всей команды;
- построить baseline-модели;
- сравнить модели по единым метрикам;
- попробовать CatBoost и подбор гиперпараметров.

### `ml_price/`

Папка предназначена для экспериментов по прогнозированию стоимости поездки.

Важное ограничение:

При реалистичном прогнозировании можно использовать только признаки, известные до начала поездки или в момент её начала. Нельзя использовать признаки, которые становятся известны только после завершения поездки.

### `reference/`

Папка со справочными материалами:

- официальный словарь данных;
- shapefile-файлы зон такси;
- географические данные, необходимые для анализа зон Нью-Йорка.

### `web/`

Папка предназначена для сайта или веб-интерфейса проекта.

Финальный интерфейс может включать:

- описание проекта;
- визуализации EDA;
- результаты моделей;
- демонстрацию прогнозирования спроса;
- демонстрацию прогнозирования стоимости;
- карты и визуализации кластеров.

## План по машинному обучению

### Прогнозирование спроса

Целевая переменная:

```text
количество поездок по зоне и часу
```

Важная логика агрегации:

- агрегировать данные по всем зонам посадки;
- агрегировать данные по всем часам;
- учитывать комбинации, где спрос равен нулю;
- не использовать `PULocationID` как обычное число, потому что числовой порядок зон не несет естественного смысла.

Возможные признаки:

- зона посадки;
- час;
- день недели;
- месяц;
- признак выходного дня;
- признак праздника;
- погодные признаки;
- спрос за предыдущий час;
- средний спрос по зоне и часу;
- район;
- координаты зоны;
- графовые признаки зоны.

Возможные метрики:

- MAE;
- RMSE;
- MAPE;
- R².

MAE особенно удобна для интерпретации, потому что показывает среднюю ошибку в количестве поездок.

### Прогнозирование стоимости

Целевая переменная:

```text
стоимость поездки
```

Важное правило:

Для реалистичной модели нужно использовать только признаки, которые известны до начала поездки или в момент её начала.

Потенциально допустимые признаки:

- время посадки;
- зона посадки;
- зона высадки, если она известна заранее;
- количество пассажиров;
- тарифный код;
- погода;
- признаки зоны;
- историческая средняя стоимость.

Потенциально недопустимые признаки:

- итоговая стоимость поездки;
- чаевые;
- дорожные сборы, если они неизвестны заранее;
- длительность поездки, если предсказание делается до завершения поездки;
- любые признаки, которые напрямую или косвенно раскрывают целевую переменную.

## Baseline-модели

Baseline-модели нужны, чтобы сравнивать более сложные модели с простыми и интерпретируемыми подходами.

Возможные baseline для прогнозирования спроса:

- глобальное среднее значение спроса;
- медианное значение спроса;
- спрос за предыдущий час;
- средний спрос по зоне посадки;
- средний спрос по зоне посадки и часу;
- линейная регрессия;
- простая tree-based модель.

Возможные baseline для прогнозирования стоимости:

- глобальная средняя стоимость;
- средняя стоимость по зоне посадки;
- средняя стоимость по паре зон посадки и высадки;
- линейная регрессия;
- Random Forest baseline.

## План по кластеризации

Кластеризацию планируется расширить: использовать не только координаты, но и дополнительные признаки.

Возможные признаки для кластеризации:

- координаты зоны;
- средний спрос;
- средняя стоимость поездки;
- средняя дистанция поездки;
- средняя длительность поездки;
- входящий и исходящий поток поездок;
- временной профиль спроса;
- район;
- графовые признаки.

Возможные метрики кластеризации:

- Silhouette Score;
- Davies-Bouldin Index;
- Calinski-Harabasz Index.

Возможные способы визуализации:

- карта кластеров;
- двумерные scatter plots;
- PCA для снижения размерности;
- профили спроса по кластерам.

## Графовые признаки

Зоны такси можно представить в виде графа.

Возможная структура графа:

- вершины — зоны такси;
- направленные ребра — поездки из зоны посадки в зону высадки;
- вес ребра — количество поездок или средний поток;
- признаки вершины — входящий поток, исходящий поток, центральность, PageRank.

Возможные графовые признаки:

- in-degree;
- out-degree;
- weighted degree;
- PageRank;
- betweenness centrality;
- спрос в соседних зонах;
- кластеры на основе потоков поездок.

Такие признаки могут помочь моделям учитывать связи между районами города, а не рассматривать каждую зону независимо.
