# New York Taxi Analysis

Проект Команды 10 в рамках Проектной Школы AI360. Мы анализируем поездки жёлтого такси Нью-Йорка, очищаем исходные данные, добавляем погодные и географические признаки, строим ML-модели и показываем результаты через веб-интерфейс.

## Цель проекта

Цель проекта — получить воспроизводимый аналитический пайплайн для данных NYC Yellow Taxi: от сырых поездок до очищенных датасетов, моделей прогноза и интерактивной визуализации. Проект отвечает на несколько прикладных вопросов:

- где и когда спрос на такси выше всего;
- какие географические зоны похожи по паттернам поездок;
- как погода и время влияют на спрос, выручку и стоимость поездки;
- насколько хорошо можно предсказывать спрос, стоимость и связанные бизнес-показатели по доступным признакам.

## Что было сделано

1. **Подготовка данных.** Проведена очистка поездок: проверены пропуски, некорректные даты, нулевые и отрицательные значения, аномальные дистанции, длительности и денежные признаки. Подготовлены сокращённые версии датасетов для хранения в репозитории и полные версии для локальной работы.
2. **Feature engineering.** К поездкам добавлены погодные признаки, календарные признаки, информация о зонах такси и географические данные из taxi zones.
3. **EDA и геоанализ.** Исследованы распределения поездок по времени, зонам, расстоянию, стоимости и погодным условиям. Подготовлены агрегированные таблицы и визуализации.
4. **Кластеризация.** Выполнены эксперименты с группировкой taxi zones по координатам и признакам поездок. Получены варианты кластеров, которые помогают интерпретировать районы Нью-Йорка по транспортному поведению.
5. **ML-модели.** Построены модели для прогнозирования спроса, стоимости поездки, выручки и времени поездки. Для стоимости сравнивались baseline-подходы, KNN, линейная регрессия и градиентный бустинг; для спроса и выручки подготовлены модели с погодными признаками.
6. **Веб-интерфейс.** Собран Streamlit-сайт с описанием проекта, визуализациями, картами и результатами моделей.

## Полученные результаты

- сформирован очищенный датасет поездок и его версии с погодными признаками;
- обучены и сохранены модели прогноза спроса и выручки с погодными признаками;
- построены модели прогноза стоимости поездки и диагностические графики качества;
- выделены кластеры зон такси и подготовлены визуализации для анализа городской структуры поездок;
- развернут веб-интерфейс проекта.

## Данные

Основной источник — данные о поездках жёлтого такси Нью-Йорка. В них есть время посадки и высадки, зоны посадки и высадки, дистанция, число пассажиров, тарифный код, тип оплаты, базовая стоимость, налоги, чаевые, дорожные сборы, дополнительные платежи и итоговая стоимость.

Дополнительно используются:

- `taxi_zone_lookup.csv` и shapefile-файлы taxi zones для связи `PULocationID` и `DOLocationID` с районами Нью-Йорка;
- погодные данные за 2023–2024 годы;

Полные датасеты не хранятся в GitHub из-за размера. Ссылка на диск будет добавлена в раздел «Ссылки».

## Структура репозитория

```text
.
├── clusterization/          # ноутбуки по кластеризации зон и паттернов поездок
├── data clean/              # очистка данных, аудит аномалий, добавление погоды
├── datasets/                # короткие датасеты, погодные CSV, lookup-таблицы, shortener.py
├── ml_demand/               # прогноз спроса, выручки и связанные ML-эксперименты
│   ├── models/              # сохранённые joblib-модели и списки признаков
│   ├── notebooks/           # загрузка, очистка, обучение моделей
│   └── reports/ tables/     # таблицы и отчётные материалы
├── ml_price/                # прогноз стоимости поездки
│   └── figures/             # графики диагностики, ошибок и важности признаков
├── reference/               # словари данных, taxi zones, презентации и справочные файлы
├── web/                     # материалы веб-части проекта
│   └── site/                # Streamlit-приложение и requirements.txt
├── check_broken_paths.py    # проверка несуществующих путей в коде и ноутбуках
└── README.md
```

## Основные папки

- `data clean/` — подготовка данных: аудит аномалий, исправление денежных признаков, объединение с погодой, формирование итоговых `.parquet`-файлов.
- `clusterization/` — эксперименты с кластеризацией taxi zones, подбор числа кластеров, анализ и визуализация результатов.
- `ml_demand/` — модели спроса и выручки по времени, зонам и погодным признакам; сохранённые модели лежат в `ml_demand/models/`.
- `ml_price/` — прогноз стоимости поездки: baseline KNN, линейная регрессия, градиентный бустинг и графики диагностики.
- `datasets/` — облегчённые версии данных для работы репозитория без полных файлов, а также погодные таблицы и lookup-файлы.
- `reference/` — официальные словари данных, shapefile taxi zones и презентационные материалы.
- `web/` — код Streamlit-сайта, который демонстрирует результаты анализа и моделей.
